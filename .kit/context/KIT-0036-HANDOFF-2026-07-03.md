## Non-obvious constraints (hard-won, don't rediscover)

- **Full plan in memory before first write** — the engine can overwrite
  itself; no disk reads of `scripts/core/` during the apply pass. There
  is a dedicated self-sync fixture test for this (D2).
- **Two-pass temp-then-commit** (KIT-0034 F3): never per-item `mv`
  inside the apply loop.
- Characterization fixtures assert **tree properties**, not console
  output — behavior gate, not gold files.
- `gh api` takes no `--repo`; inline owner/name in the tarball path.
  Tarballs unpack into a nested `<owner>-<repo>-<sha>/` root —
  `resolve_source` normalizes this; the engine never sees it.
- The exit-code contract (0/1/2/3/4) is frozen by tests once merged —
  it went through review; don't improvise extra codes. Wrapper UX
  branches on `SyncReport.status`, not exit codes.
- DK rules apply: `encoding=` on `open()`, no bare `except`, `==` for
  identifier comparison. Run `pattern_lint.py` on new Python.
- Never stage or delete `.kit/adversarial/` (user-owned, untracked).
