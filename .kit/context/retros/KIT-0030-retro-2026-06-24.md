## What went well

- **Diverse-provider evaluator review paid off.** Three model families (o3,
  Gemini-3-flash, Mistral-large) all returned REVISION_SUGGESTED with
  *convergent* findings — strong signal the gaps were real, not model noise.
  Worth doing for any non-trivial ADR.
- **The pre-mortem changed the design.** Imagining failure scenarios surfaced
  that both the push (control-drift) and pull (harvest-drift) models depend on
  the same historically-weak link — backflow/upstreaming — which reframed the
  whole problem.

## What to do differently

- **Confirm the actual scope before designing.** I designed a multi-tenant
  control system (schema + render + three-way version handshake) when the real
  need was "keep *my own* projects in sync." The user's one-line clarification
  ("my own projects", "maybe just an upgrader agent?") collapsed most of the
  ADR's machinery. Ask "who is this for and how many?" up front — it changes
  the answer by an order of magnitude.

## Decisions / direction

- For a single operator, the upgrade path is an **upgrader agent** that wraps
  the guide and applies judgment only on the fuzzy steps (preserve local
  adaptations, rewrite `model:` pins). Mechanical axes call existing
  deterministic tooling (`/plugin update`, manifest sync).
- KIT-ADR-0025 is **Proposed** but its heavy schema/render machinery is likely
  unnecessary at this scale; a slim revision ("upgrader agent + the guide as
  the lightweight convention") is expected. Not yet done.

## Reusable gotchas confirmed (cross-session)

- **Marketplace must be GitHub-sourced, not a local directory.** A `directory`
  source (spike leftover) serves disk, defeating pins. `marketplace update`
  alone won't re-point it — `remove` + `add` is required.
- **Pre-commit hook aborts the commit when it reformats a file** (end-of-file
  fixer). Re-stage the hook's change and commit *fresh* — never `--amend` (the
  commit didn't happen).
- **CI pytest 9.1.x vs local 9.0.x** diverges on logging: pytest 9.1.x attaches
  a capture handler that trips a naive `if logger.handlers` guard. Tag your own
  handlers.
- **Classifier hard-blocks** the agent from editing its own `settings.json` and
  from pushing to repos outside the trusted org. Those are user actions.
